# 🧬 微生物帝国 — 节奏×连线 融合设计文档

> **版本**: v1.0 | **日期**: 2026-03-21 | **状态**: 设计提案
> **替代**: PACING_ENHANCEMENT.md v1.0（节奏部分重新设计，砍掉纯等待机制）
>
> **核心理念**：用"乐趣"填满"等待" — 不强制空等，而是解锁新玩法让玩家自然花时间

## §0 设计背景与问题诊断

### 两个玩家反馈

1. **"内容太少，很快到终局"** — 首通 ~50-70分钟，玩家感觉"吃太快"
2. **"连线缺乏乐趣"** — 对比 Factorio/Shapez 等工厂游戏，当前传送带缺少策略深度

### 前序方案的反思

| 前序方案 | 问题 |
|---------|------|
| PACING_ENHANCEMENT.md §1 进化培养30-150秒纯等待 | "注水"——拉长时间但没增加决策或操作 |
| §3 科技研究时间×2 | 纯拉时间——玩家体验是"看进度条" |
| 连线方案①距离衰减（单独做） | 纯debuff——惩罚远距离但没给奖励 |

**核心教训**：拉长时间 ≠ 增加深度。玩家要的是"更多好玩的事情做"，不是"更多等待"。

### 融合思路

> ~~"强制等待 → 然后放行"~~
> **"解锁新的好玩的东西 → 玩家自然花时间 → 顺便就够条件升级了"**

### 当前系统现状（代码依据）

| 系统 | 现状 | 代码位置 |
|------|------|---------|
| 手动传送带最大距离 | **4格曼哈顿距离** | `handleBeltConnectClick` L12843: `dist > 4` |
| 虚拟邻接 | 非物理邻居的传送带连接获 **50%** 邻接加成，最多 **2条** | L6828-6863, `MAX_VIRTUAL = 2` |
| 端口系统 | 每种建筑有 maxIn/maxOut 限制 | `PORT_DEFS` L626-653 (22项) |

| 传送带模式 | P1=自动, P2=混合, P3+=纯手动 | `BELT_MODE` L711-715 |
| 传送带等级 | 1-5级，升级改善效率和容量 | L3098-3101 |
| 连击系统 | 4秒内连续连线触发连击奖励 | `addBeltCombo` L7062-7087 |

## §1 进化培养任务（替代纯等待）

### 问题

原方案 §1 的进化培养是"点击 → 等N秒 → 完成"。**培养期间没有新决策**——建筑照常运转，玩家只能看进度条。这是假深度。

### 方案：3选1培养任务

点击进化时，不再等待计时器。弹出 **3个随机微任务**，玩家选择并完成1个即可进化。

#### 任务池设计

```
🧬 进化培养中 — 完成一个培养条件：

  [A] 🔗 连线任务    (操作类 — 和传送带系统联动)
  [B] 🏗️ 布局任务    (规划类 — 和邻接系统联动)
  [C] ⏱️ 经济任务    (管理类 — 和资源系统联动)
```

每次进化从当前阶段的任务池中随机抽取3个（每类各1个），玩家选择最想做的那个。

#### 各阶段任务池

**P1→P2 进化（简单操作，目的：教学）**

| 类型 | 任务 | 目的 |
|------|------|------|
| 🔗连线 | 建立2条传送带连接 | 引导使用传送带 |
| 🏗️布局 | 发现1条新邻接规则 | 引导关注布局 |
| ⏱️经济 | 维持30秒葡萄糖正收益 | 感受经济平衡 |

**P2→P3 进化（引入远距离/效率概念）**

| 类型 | 任务 | 目的 |
|------|------|------|
| 🔗连线 | 建立1条**2格以上**距离的传送带 | 引导发现远距连线 |
| 🏗️布局 | 发现3条新邻接规则 | 深入邻接系统 |
| ⏱️经济 | 同时拥有4种不同资源≥10 | 资源多样化管理 |

**P3→P4 进化（策略深度）**

| 类型 | 任务 | 目的 |
|------|------|------|
| 🔗连线 | 达成3连击传送带连接 | 展示连击系统 |
| 🏗️布局 | 使1个建筑获得≥20%邻接加成 | 精心布局回报 |
| ⏱️经济 | 让1个多输入建筑达到供给同步加成 | 引导同步系统 |

**P4→P5 进化（综合挑战）**

| 类型 | 任务 | 目的 |
|------|------|------|
| 🔗连线 | 建立1条4格距离的传送带连接 | 远距物流极限 |
| 🏗️布局 | 建造并使用中继节点（见§5） | P3b核心建筑 |
| ⏱️经济 | QS信号库存≥15 | P4资源管理 |

#### 跳过机制

- 任务弹出后 **60秒内完成**：正常进化 + 🎉额外奖励（少量资源）
- **60-120秒**：正常进化，无额外奖励
- **120秒后**：可以支付额外资源直接跳过（`[跳过：消耗20🧬]`）
- 转生玩家可解锁"快速培养"减免：跳过成本-50%

#### 机制规格

```
目的：让进化过程成为主动体验而非被动等待
玩家幻想：进化是一次有准备的"蜕变"，不是攒够资源按按钮
输入：玩家点击"进化"按钮
输出：弹出3选1任务面板 → 完成任务 → 进化生效
成功条件：>90%玩家在60秒内完成任务（首通），>80%觉得"有趣不繁琐"
失败状态：玩家反复选择跳过 → 说明任务太难/太烦 → 降低难度阈值
边界情况：
  - 如果玩家已经满足某个任务条件了（如已有2条传送带）？→ 直接判定完成，秒过
  - 如果玩家在完成任务前回收了关键建筑？→ 任务条件实时检测，回收后重新计数
  - 3个任务都觉得太难？→ 120秒后可跳过，不会永远卡住
调整杠杆：任务难度、跳过等待时间、跳过资源成本、额外奖励数值
依赖关系：进化系统、传送带系统、邻接系统、资源系统
```

#### 预期效果

- 每次进化增加 **15-45秒主动操作**（而非空等）
- 引导玩家使用传送带、邻接、经济系统（教学功能）
- 总进化操作时间增加约 **2-4分钟**（4次进化×30-60秒平均）

## §2 阶段探索度（连线深度整合版）

### 问题

原方案 §2 的探索度是好框架，但积分项偏被动（"拥有X个建筑"、"研究Y科技"）。需要让**连线和布局操作**成为核心积分来源——引导玩家"为了升级去玩连线，结果发现连线本身很好玩"。

### 方案：探索度 = 连线 + 布局 + 系统的综合积分

保持"需要N分探索度才能推进"的框架，但重新设计积分项，**约40%来自连线/布局操作**。

## §3 科技研究前置条件（替代时间膨胀）

### 问题

原方案 §3 将科技研究时间×2 是纯注水。玩家体验从"等20秒"变成"等40秒"——没有新决策，只有更长的进度条。

### 方案：科技研究需要前置操作条件

研究时间**不变**。但部分科技需要先完成一个操作条件才能开始研究。

```
原来: 花资源 → 点研究 → 等20秒 → 完成
现在: 满足条件 → 花资源 → 点研究 → 等20秒(不变) → 完成
```

#### 前置条件表

| 科技 | 研究时间(不变) | 新增前置条件 | 设计意图 |
|------|--------------|------------|---------|
| 纯培养技术 | 15s | 无（P1入门科技不设门槛） | — |
| 高效采集 | 20s | 拥有3+传送带连接 | 引导早期传送带 |
| 快速代谢 | 20s | 拥有2台ATP合成酶 | 引导建筑投资 |
| 基础代谢学 | 25s | 发现3条邻接规则 | 引导邻接探索 |
| 氮循环工程 | 30s | 拥有1条2格+距离传送带 | 引导远距连线 |
| 蛋白质工程 | 30s | 完成1个培养皿实验 | 引导培养皿使用 |
| 生物膜技术 | 35s | 手动连接5+传送带 | P3物流准备 |
| 自适应物流 | 30s | 拥有2条2格+距离传送带 | 远距物流铺垫 |
| 共生网络 | 35s | 发现8条邻接规则 | 深度布局 |
| 群体感应 | 40s | 2个建筑达到供给同步 | 同步系统掌握 |

转生加成"科研捷径"在新系统中效果：
- 研究时间仍然 -30%（基于原研究时间）
- 前置条件的数值要求 **-25%**（向下取整，最低1）
  - 例："5+传送带" → 转生后降为"4+传送带"

#### 预期效果

- 不增加研究等待时间（**0秒注水**）
- 引导玩家在研究前自然完成2-5分钟的操作
- 每个前置条件恰好是"你本来应该做但可能忽略的事"

## §4 距离-效率衰减 + 远距成就（连线策略深度）

### 问题

当前4格内连线没有距离代价。所有距离的连线效果相同——没有"选择"存在。

### 方案：距离衰减 + 首次远距奖励

#### 效率衰减曲线

| 曼哈顿距离 | 传输效率 | 维护成本 | 视觉表现 |
|-----------|---------|---------|---------|
| 1（相邻） | **100%** | 免费 | 粗实线 + 密集粒子 |
| 2 | **92%** | 免费 | 实线 + 正常粒子 |
| 3 | **80%** | 0.3⚡/s | 细实线 + 稀疏粒子 |
| 4 | **65%** | 0.6⚡/s | 虚线 + 稀疏粒子 |

> 公式: `efficiency = 1.0 - 0.04 * (dist - 1)^1.6`
>
> 维护费: `dist >= 3 ? 0.3 * (dist - 2) : 0`

#### 首次远距成就系统（消除纯debuff感）

**关键设计**：距离越远效率越低是策略代价，但**第一次**达成远距离连线给予奖励和成就感。

| 触发条件 | 成就名 | 奖励 | 弹出文案 |
|---------|--------|------|---------|
| 首次2格连线 | 🔗 远程传输 | +10⚡ + 探索度+1 | "突破了物理限制！" |
| 首次3格连线 | 🔗 远程物流 | +20⚡ + 3🟢 + 探索度+1 | "化学信号跨越三格！" |
| 首次4格连线 | 🔗 物流大师 | +30⚡ + 5🟢 + 探索度+1 | "极限距离的精密控制！" |
| 累计10条远距(2格+) | 🔗 网络编织者 | +50⚡ + 10🧬 | "整个培养皿都是你的网络" |

#### 虚拟邻接联动

当前虚拟邻接（传送带连接的非物理邻居获50%邻接加成）与距离衰减叠加：

```
最终传输效率 = 基础效率(距离) × 传送带等级效率 × 端口效率折扣
虚拟邻接加成 = 匹配规则加成 × 50% × (不受距离衰减影响)
```

**设计决策**：虚拟邻接加成**不受距离衰减影响**——这意味着远距离连线虽然传输效率低，但邻接加成不打折。玩家会发现"远距连一条到特定建筑虽然传输差，但邻接加成很赚"——增加了布局策略深度。

#### 调整杠杆

| 变量 | 基础值 | 范围 | 调整备注 |
|------|--------|------|---------|
| 效率衰减指数 | 1.6 | 1.2-2.0 | 越高则远距惩罚越重 |
| 距离2效率 | 92% | 88-96% | [占位符] - playtest后调整 |
| 距离3维护费 | 0.3⚡/s | 0.1-0.5 | 不宜过高，P3玩家ATP不富裕 |
| 距离4维护费 | 0.6⚡/s | 0.3-1.0 | [占位符] |

#### 预期效果

- 玩家需要在"紧凑布局(高效但拥挤)"和"松散布局(灵活但效率税)"之间决策
- 首次远距成就提供正反馈，消除"被惩罚"感
- 和邻接系统形成协同策略——远距连线打开新的邻接组合可能性

## §5 中继节点（P3b核心新建筑）

### 定位

信号中继站是 **P3b** 的标志性新建筑，引入"中转物流"玩法。不只是"又解锁了一个建筑"，而是"整个连线方式变了"。

### 建筑规格

```
建筑: 信号中继站 (signalRelay)
阶段: P3b
类型: 物流辅助（不生产/不消耗资源）

成本: 8🧪 + 5🧱
解锁条件: P3b 子阶段 (见§6)

端口: maxIn: 3, maxOut: 3
特殊: 中继模式 — 输入的资源自动转发到输出端

主题化: "化学信号中继" — 微生物通过扩散中继传递远距离信号
SVG: 📡 带辐射波纹的简约符号
```

### 核心机制

#### 1. 距离延伸

```
A → 中继 → B

A到中继: 最远4格
中继到B: 最远4格
总距离:  最远8格（单次连线的2倍）
```

#### 2. 中继不衰减

中继站之间（或中继站到建筑）的连线，**距离衰减减半**：

```
普通连线:  dist=3 → 80%效率
经过中继:  dist=3 → 90%效率  (衰减减半)
```

#### 3. 中继链

多个中继站可以串联：

```
A → 中继1 → 中继2 → B  = 最远12格
```

但每经过一个中继站，有 **5%固定效率损耗**：

```
A→中继1(4格,90%效率) → 中继2(4格,90%×95%=85.5%) → B(4格,85.5%×95%×90%=73.1%)
```

**设计意图**：中继链有用但有代价——玩家要权衡"多跳转中转"和"直连但距离限制"。

#### 4. 中继节点邻接规则

| 规则名 | 自身 | 邻居 | 加成 | 说明 |
|--------|------|------|------|------|
| 中继共振 | signalRelay | signalRelay | +12% | 中继站互相增幅 |
| 膜下中继 | signalRelay | biofilmReactor | +8% | 生物膜保护信号 |
| 菌丝中继 | signalRelay | myceliumNet | +10% | 菌丝网络是天然中继 |
| 孢子中继 | signalRelay | sporeSpreader | +6% | 孢子携带信号 |

#### 5. 中继节点与供给同步

中继节点转发的资源也参与供给同步计算。如果一个建筑通过中继接收到的多种资源都平衡，仍然触发同步加成。

#### 机制规格

```
目的：增加物流空间规划深度，让P3阶段有标志性新玩法
玩家幻想：像搭建通讯网络一样规划物流路线
输入：放置中继站 → 从源建筑连线到中继 → 从中继连线到目标
输出：远距离资源传输通道建立
成功条件：>70%首通玩家在P3b使用至少1个中继站
失败状态：玩家完全无视中继站 → 可能需要调低直连距离限制or增加中继独占连线奖励
边界情况：
  - 中继站输入满了但输出还有空？→ 资源暂存在中继(小缓冲)，满后堵塞
  - 玩家把中继当"免费建筑"刷邻接？→ 中继有成本(8🧪+5🧱)，数量受格子限制
  - 中继链无限延伸？→ 每跳5%固定损耗，超过3跳实际效率很低
调整杠杆：中继衰减减半系数、每跳固定损耗%、端口数、建造成本
依赖关系：传送带系统、端口系统、距离衰减系统(§4)、邻接系统
```

### 问题

P3拥有最多的内容（7建筑、4配方、变异实验室）但玩家往往匆匆通过。

### 方案：P3 = P3a + P3b

| **P3b** 物流进阶 | P3a探索度≥3 | 代谢回路、**信号中继站**(新)、发酵液泡、生物质转化炉 |

**P3a→P3b 探索度（需3分，从以下任选3）：**

| 行为 | 分值 | 引导 |
|------|------|------|
| 拥有2种P3a建筑 | +1 | 基础建设 |
| 完成1个P3培养皿实验 | +1 | 配方探索 |

**P3b 带来的核心变化**：

- 两步走设计让P3有两次"新玩具"的惊喜感

## §7 连线可视化增强

### 问题

当前传送带渲染是Canvas L路径+粒子。信息密度低——看不出"这条线效率高不高""哪里是瓶颈"。

### 方案

#### 7.1 传输量可视化

| 传输状态 | 粒子密度 | 粒子速度 | 线条粗细 |
|---------|---------|---------|---------|
| 满载(≥80%) | 密集 | 快 | 3px |
| 正常(40-80%) | 正常 | 中 | 2px |
| 低载(<40%) | 稀疏 | 慢 | 1.5px |
| 空闲(0%) | 无粒子 | — | 1px虚线 |

#### 7.2 瓶颈高亮

当一条传送带成为供应链瓶颈时（接收端需求 > 传送带实际吞吐）：
- 传送带闪烁 **红色脉冲**（0.8秒周期）
- 鼠标悬浮显示："⚠️ 瓶颈：需求 2.5/s，实际 1.8/s"
- 在传送带中点位置显示 ⚠️ 图标

#### 7.3 距离标识

远距离传送带（2格+）在连线中点显示距离数字：

```
[建筑A] ────── ②  ────── [建筑B]    ← 2格距离
[建筑A] ──── ③ ──── [中继] ──── ② ── [建筑C]
```

#### 7.4 网络总览模式（可选，P1优先级）

按 `Tab` 键切换到简化视图：
- 建筑显示为小圆点（颜色=阶段）
- 传送带显示为直线（颜色=效率，绿→黄→红）
- 中继节点显示为大圆圈
- 用于快速审视整体物流拓扑

#### 预期效果

- 传输量可视化让连线"有生命感"——满载的传送带像血管一样脉动
- 瓶颈高亮引导玩家优化物流——"哦这里堵了，该升级传送带或加中继"
- 距离标识帮助判断效率衰减代价

## §12 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 培养任务太简单，秒过无感 | 进化仍然太快 | 阶段递进难度 + 条件实时检测（已满足的不重复给） |

| 系统间耦合太紧 | 改一个系统连锁反应 | 各系统通过"探索度积分"松耦合，非直接依赖 |
