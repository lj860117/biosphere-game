# 社交功能增强设计方案 v1.0

> 微生物帝国 — 排行榜 / 分享 / 成就展示增强

---

## 📊 现状审计

### 已有功能清单
| 功能 | 状态 | 问题 |
|------|------|------|
| Supabase排行榜(当世/传奇) | ✅ 完整 | UI朴素,只有分数排名 |
| 迷你排行Top3 | ✅ 完整 | 信息密度低,只有名字+分数 |
| 排名头衔系统(4档) | ✅ 完整 | 仅文字显示 |
| 昵称/帝国名 | ✅ 完整 | — |
| 文字分享卡片 | ✅ 基础 | **纯文本**, 无视觉吸引力 |
| 成就殿堂弹窗 | ✅ 完整 | 仅自己看,无社交维度 |
| 成就里程碑buff | ✅ 完整 | — |
| 文明编年史 | ✅ 完整 | 仅本地展示,不可分享 |

### 核心缺失
1. **视觉分享** — 当前只能复制文字卡片，无法生成截图/图片
2. **对比维度** — 排行榜只看分数，无法看到别人的"帝国全貌"
3. **成就社交** — 30+个成就仅自己欣赏，无稀有度/炫耀感
4. **竞争动力** — 没有"追赶目标"或"被超越警告"
5. **轻社交** — 没有低门槛的互动方式（点赞/喝彩）

## 🎯 设计方案：三个增强模块

### 模块 A：帝国分享卡（视觉截图）
**核心理念**：把纯文本卡片升级为可视化的"帝国名片"，用 Canvas 渲染成图片

#### A1. 分享卡内容设计
```
┌────────────────────────────────┐
│  🧫 微 生 物 帝 国            │  ← 品牌标题
│  ─────────────────────         │
│  「{帝国名}」                   │  ← 玩家帝国名
│  {称号} · {排名头衔}            │  ← 菌主称号 + 排行头衔
│                                │
│  ┌──────┐  🏆 {总分} [{评级}]  │  ← 左侧培养皿缩略图
│  │ 培养 │  📡 阶段 P{N}        │     右侧核心数据
│  │ 皿   │  🧬 进化 Lv.{N}      │
│  │ 缩略 │  🏗️ 建筑 {N}座       │
│  │ 图   │  🦠 峰值 {N}         │
│  └──────┘  ⏱ 在线 {时间}       │
│                                │
│  ─────────────────────         │
│  🏆 成就 {N}/{Total}           │  ← 成就进度条
│  ★★★☆☆ 稀有成就: {3个图标}     │  ← 精选稀有成就
│                                │
│  🧫 微生物帝国 · biosphere.game│  ← 品牌水印
└────────────────────────────────┘
```

#### A2. 技术方案
- **Canvas 原生渲染** — 不依赖 html2canvas 等库
- 培养皿缩略图：截取当前 `.grid` 区域为迷你图（每格2-3px，色块表示建筑类型）
- 背景：阶段对应的渐变色 + 品牌微粒纹理
- 输出：`canvas.toBlob()` → 复制到剪贴板 / 下载PNG
- **Web Share API** 优先尝试（移动端直接调起微信/QQ分享），降级到下载

#### A3. 分享触发点
- 排行榜弹窗内新增「📸 分享名片」按钮
- 阶段转换仪式结束后的「🎉 分享此刻」按钮
- 文明编年史底部的「📸 分享编年史」按钮
- 成就殿堂的「📸 炫耀成就墙」按钮

### 模块 B：排行榜增强 — 对比、追赶、被超越

#### B1. 排行榜卡片化
**现状**：每行只有 `排名 | 名字 | 分数 | 评级 | 阶段/建筑`

**增强为双行卡片**：
```
┌──────────────────────────────────────┐
│ 🥇 菌落A帝国          S  2.4M 分    │  ← 主行: 排名+名字+评级+分数
│    P5 · 28🏗 · 22🏆 · ☀️完  · 3♻️    │  ← 副行: 阶段·建筑·成就·奇观·转生
└──────────────────────────────────────┘
```

新增字段（Supabase 表新增列）：
- `achievements_count` — 成就数量
- `prestige_count` — 转生次数
- `online_time` — 在线时长
- `rare_badges` — 稀有成就ID列表（JSON）

#### B2. "追赶目标" 机制
- 在迷你排行下方显示：`📌 距 #{N} {名字} 还差 {分数差}`
- 当我的分数接近上方玩家（差距<15%）：`🔥 即将超越 {名字}！`
- 配合轻微脉冲动画引起注意

#### B3. "被超越" 推送
- 每次刷新排行榜时，对比上次缓存的排名
- 如果排名下降：`⚠ {名字} 超越了你！你现在第 {N} 名`
- 用 milestone toast 展示，非阻断性

#### B4. 排行榜分榜
除了"当世/传奇"双榜外，新增：
- **速度榜** — 按到达P5用时排序（鼓励speedrun）
- **效率榜** — 按全局效率%排序（鼓励优化布局）
- **成就榜** — 按成就完成数排序（鼓励探索）

用Tab切换，复用现有UI框架。

### 模块 C：成就社交增强

#### C1. 成就稀有度标签
根据全服解锁率给成就打标：
- 💠 **传说** (<5% 玩家解锁) — 紫色发光
- ⭐ **稀有** (5-20%) — 金色
- ✦ **罕见** (20-50%) — 银色
- 无标签 (>50%) — 普通

**实现**：Supabase 表新增 `achievement_stats`，定期统计各成就的解锁率。客户端每次拉排行榜时顺带拉取。

#### C2. 成就展柜（Profile Badge）
玩家可从已解锁成就中选择 **3个** 作为"展柜"：
- 展柜成就显示在排行榜名字后面（3个emoji徽章）
- 展柜成就显示在分享卡片中
- 在迷你排行中名字后也带展柜标

#### C3. 成就解锁广播
当有人解锁 **金/钻石** 级成就时，在全服玩家的日志区显示：
```
🌟 {帝国名} 解锁了「{成就名}」！
```
- 只推送金+钻石级，避免刷屏
- 使用 Supabase Realtime 订阅
- 展示在日志栏顶部，3秒后消失

#### C4. 首通成就
特定成就记录全服"第一个解锁的人"：
- 在成就殿堂中显示：`🏴 首通: {帝国名}`
- 永久记录在 Supabase
- 适用于：戴森球完成、极速奇观、传奇霸主等终极成就

## ⚙️ 后端变更（Supabase）

### 表结构扩展
```sql
-- leaderboard 表新增列
ALTER TABLE leaderboard ADD COLUMN achievements_count INT DEFAULT 0;
ALTER TABLE leaderboard ADD COLUMN prestige_count INT DEFAULT 0;
ALTER TABLE leaderboard ADD COLUMN online_time INT DEFAULT 0;
ALTER TABLE leaderboard ADD COLUMN showcase_badges TEXT DEFAULT '[]';  -- JSON: 3个成就ID
ALTER TABLE leaderboard ADD COLUMN fastest_p5 INT DEFAULT NULL;  -- 到达P5秒数

-- 新表: achievement_stats (定期统计)
CREATE TABLE achievement_stats (
  id TEXT PRIMARY KEY,           -- 成就ID
  unlock_count INT DEFAULT 0,    -- 解锁人数
  total_players INT DEFAULT 0,   -- 总玩家数
  first_player TEXT DEFAULT NULL, -- 首通玩家名
  first_time TIMESTAMPTZ DEFAULT NULL,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 新表: achievement_broadcasts (实时广播)
CREATE TABLE achievement_broadcasts (
  id BIGSERIAL PRIMARY KEY,
  player_name TEXT NOT NULL,
  achievement_id TEXT NOT NULL,
  achievement_name TEXT NOT NULL,
  tier TEXT NOT NULL,  -- gold/diamond
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Supabase Edge Function
- `update-achievement-stats`: CRON每小时运行，统计各成就解锁率
- `broadcast-achievement`: 插入broadcast表 + Realtime推送

## 🖼️ 分享卡视觉规格

### 尺寸
- **标准**: 800×600px (适合微信/QQ发送)
- **方形**: 800×800px (适合朋友圈/Instagram)
- 用户可选(默认标准)

### 色彩
- 背景跟随当前阶段色调:
  - P1: 深蓝绿 → 青 渐变
  - P2: 青 → 蓝绿 渐变
  - P3: 蓝绿 → 金黄 渐变
  - P4: 金黄 → 紫 渐变
  - P5: 紫 → 品红 渐变
- 品牌水印: 半透明白色
- 数据区: 半透明暗底卡片

### 培养皿缩略图渲染
- 每格 → 对应建筑类型色块(3×3px → 5×5px 取决于网格大小)
- 核心格: 发光中心点
- 空格: 极暗色
- 供能链接: 淡色连线
- 生物膜: 半透明外圈

## 💡 设计哲学

**社交功能的三层漏斗**：

1. **被动社交（零门槛）** — 排行榜看别人的帝国数据,产生"我也要"
2. **轻量分享（低门槛）** — 一键截图发给朋友,朋友看到想玩
3. **竞争驱动（持续参与）** — 追赶/被超越/首通, 给中后期玩家持续动力

核心目标不是做社交网络，而是**让玩家的成长有观众，让孤独的放置游戏有人气感**。
