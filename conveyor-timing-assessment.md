# 🏭 传送带时序同步玩法评估 — 异星工厂模式可行性分析

> 用户问题：能否加入类似异星工厂（Factorio）的"传送带长度影响到达时间、多路资源同时到达才能持续产出"的玩法？  
> 与已有邻接系统和上次评估的方案能否结合？

---

## 一、先理解两种根本不同的资源模型

### 异星工厂（Factorio）的模型：离散物品流

```
铁矿石(物品) → [传送带 3秒] → 冶炼炉 → [传送带 1秒] → 组装机
                                              ↑
铜矿石(物品) → [传送带 5秒] ──────────────────┘

关键：组装机需要1个铁板+1个铜板"同时在手"才开始一个周期
      如果铜板的传送带太长，组装机就会"等铜板"，停顿。
      物品是离散的：一个一个走在传送带上。
```

### 微生物帝国的当前模型：连续速率流

```
碳源采集器 → [传送带] → ATP合成酶
  prod: 1.5 glucose/s       cons: 1 glucose/s

关键：updateRates() 每帧计算全局速率
      beltMult = getBeltMultiplierForBuilding(idx)
      最终产出 = 基础产出 × 效率 × beltMult × adjBonus × ...
      
      传送带影响的是"乘数"（效率/容量），不是"到达时间"
      资源是连续的：每秒产出 X 单位，不是一个一个走
```

### 这意味着什么？

| 维度 | Factorio（离散物品） | 微生物帝国（连续速率） |
|------|---------------------|---------------------|
| 传送带上有什么 | 独立的物品实体，有位置 | 视觉粒子（纯装饰） |
| 产出计算 | 物品到达→触发一次合成 | 每帧按速率累加 |
| 长度的影响 | 决定延迟时间 | 影响效率乘数（距离>4格无法连接） |
| "阻塞"的含义 | 传送带满了，上游停产 | `capFactor = min(1, totalCap/totalDemand)` |
| 同步问题 | 物理存在（两条带同时送到） | 不存在（速率是瞬时的） |

## 二、直接移植 Factorio 时序模型的代价

### 如果硬改成离散物品模型

**需要改什么：**

```
1. 资源系统重写
   - 当前: res.glucose += rate * dt（全局池）
   - 改为: 每个建筑有独立的输入/输出缓冲区
   - 改动量: ~500行，涉及 updateRates() 整体重构

2. 传送带系统重写  
   - 当前: 传送带是"乘数修正器"
   - 改为: 传送带是"物品传输管道"，有容量、延迟、排队
   - 改动量: ~400行，computeBelts() + getBeltMultiplierForBuilding() 完全重写

3. 粒子系统从装饰变为实体
   - 当前: drawConveyorParticles 是视觉效果
   - 改为: 每个粒子代表真实资源，位置=真实位置
   - 改动量: ~300行

4. UI 重写
   - 资源面板从"速率"变为"存量+吞吐量"
   - 建筑面板增加缓冲区显示
   - 改动量: ~200行
```

**总改动量：~1400 行，相当于重写 13% 的代码**

**风险评级：☠️ 致命**
- 破坏所有已有的平衡数据（每个建筑的 prod/cons 比例全部失效）
- 破坏邻接系统（邻接加成是速率乘数，离散模型下没有对应物）
- 破坏培养皿系统、科技树、转生系统等所有与速率挂钩的子系统
- 存档兼容性归零
- 玩法体验从"悠闲放置"变为"精确工程"，目标受众完全改变

### ❌ 结论：直接移植不可行，也不应该做

## 三、真正适合微生物帝国的方案：在速率模型上增加"时序协同"

### 核心思路：不改底层模型，在现有速率计算上叠加一个"同步加成"层

**灵感转化**：Factorio 的时序同步本质是什么？
→ **多条输入线路同时饱和时，产出最优**
→ 这个"同时饱和"在速率模型中可以翻译为：**所有输入传送带的容量利用率同时达到高水平**

---

### 🏆 方案 F：供给链同步加成（Supply Chain Synchronization Bonus）

**设计原理**：

```
对于多输入建筑（如蛋白质工厂需要氮源+能量、生物膜需要葡萄糖+氮+能量）：

计算每条输入传送带的"供给充裕度":
  充裕度 = 该传送带实际输入速率 / 该资源的需求速率

如果所有输入的充裕度都 ≥ 某个阈值 → 触发"同步加成"
```

**具体实现**：

```javascript
// 在 getBeltMultiplierForBuilding() 基础上，新增同步加成计算
// 不修改现有逻辑，纯粹叠加

getSyncBonus(idx) {
  const g = this.grid[idx];
  const bd = BLDS[g.type];
  if (!bd || Object.keys(bd.cons).length < 2) return 0; // 只有多输入建筑才有同步加成
  
  // 统计每种输入资源的供给充裕度
  const inputTypes = Object.keys(bd.cons);
  const adequacies = []; // 每种资源的充裕度 0~1+
  
  for (const resKey of inputTypes) {
    const demand = bd.cons[resKey];
    // 找为这个建筑输送该资源的传送带
    const supplyRate = this.getInputRateForResource(idx, resKey);
    adequacies.push(Math.min(supplyRate / demand, 1.5)); // cap 1.5
  }
  
  if (adequacies.length < 2) return 0;
  
  // 同步度 = 所有充裕度的最小值（木桶效应）× 均衡度（方差越小越好）
  const minAdequacy = Math.min(...adequacies);
  const avgAdequacy = adequacies.reduce((a,b) => a+b, 0) / adequacies.length;
  const variance = adequacies.reduce((s, v) => s + (v - avgAdequacy) ** 2, 0) / adequacies.length;
  const balance = 1 / (1 + variance * 4); // 方差越小，均衡度越接近1
  
  // 同步加成 = 最低充裕度 × 均衡度 × 基础系数
  // 所有输入都饱和且均衡 → 最高 +25% 加成
  // 某条输入不足或严重不均衡 → 加成接近 0
  const syncBonus = minAdequacy * balance * 0.25;
  
  return Math.max(0, syncBonus);
}
```

**加成接入点**（只需在 updateRates 的 mult 计算中追加一项）：

```javascript
// 现有代码（行5768）：
const mult = this.gEff * popMult * bldMult * beltMult * techBonus * adjBonus * ...;

// 改为：
const syncBonus = this.getSyncBonus(idx);
const mult = this.gEff * popMult * bldMult * beltMult * techBonus * adjBonus * (1 + syncBonus) * ...;
```

**视觉反馈**：

```
同步度 < 30%  → 无特殊效果
同步度 30~60% → 建筑格子微微发光（浅金色边框）
同步度 60~90% → 金色呼吸光 + 所有输入传送带显示"⚡同步"标记
同步度 > 90%  → 强烈的金色脉冲 + "⚡完美同步！+XX%" 浮动数字
```

**改动量：~60 行 JS（核心计算）+ ~30 行 JS（视觉）= ~90 行**

### 为什么这个方案好？

| 维度 | 评价 |
|------|------|
| **保留速率模型** | 完全不动现有 updateRates 架构 |
| **不冲突邻接系统** | 邻接=相邻加成，同步=供给均衡加成，正交维度 |
| **传送带更有意义** | 不再只是"有vs没有"，而是"供给是否均衡" |
| **鼓励规划** | 多输入建筑需要确保每种资源的传送带都到位 |
| **直观可感** | "完美同步+25%"是非常清晰的正反馈 |
| **Factorio 精髓** | 捕捉了"同时到达才高效"的核心乐趣，不是皮毛 |

## 五、与邻接系统的关系 — 互补而非冲突

### 三维布局优化矩阵

```
维度1：邻接系统 → 奖励"放在哪里"（空间位置）
维度2：传送带系统 → 奖励"连了什么"（网络拓扑）  
维度3：同步加成 → 奖励"连得多均衡"（供给质量）
```

| 优化维度 | 邻接 | 传送带 | 同步 | 最优策略 |
|---------|------|--------|------|---------|
| 蛋白质工厂 | 紧邻固氮(+12%) | 连接固氮+能量 | 氮+能量均衡(+25%) | 固氮在旁、能量管线长度匹配 |
| 生物膜反应器 | 紧邻碳源+固氮+能量 | 三条输入管线 | 三路均衡(+25%) | 中心位置，三个方向放原料建筑 |

**关键洞察**：邻接系统鼓励"紧挨着放"，同步系统鼓励"多路供给均衡"。两者可能产生有趣的张力——

> 你可能想把蛋白质工厂紧邻固氮装置（邻接+12%），但如果能量站太远，
> 同步度就低。最优解是找到一个**三方都能兼顾的布局**——这才是真正的工程规划乐趣！

### 不冲突的原因

```
邻接加成：adjBonus = 1 + 邻接规则累加     （只看四方格子）
传送带乘数：beltMult = 效率 × 容量因子     （看连接是否存在）
同步加成：syncBonus = 供给均衡度 × 0.25    （看多输入的平衡性）

三者在 updateRates 的 mult 公式中各占独立位置，互不干扰。
```

## 六、哪些建筑会受益于同步加成？

根据 BLDS 数据，**多输入建筑**才有同步加成：

| 建筑 | 消耗（多输入） | 同步目标 | 最大同步加成 |
|------|---------------|---------|------------|
| simpleExtractor | glucose + energy | 碳源+能量均衡 | +25% |
| proteinFactory | nitrogen + energy | 氮+能量均衡 | +25% |
| geneExtractor | protein + energy | 蛋白+能量均衡 | +25% |
| aminoSynth | nitrogen + glucose | 氮+碳均衡 | +25% |
| ribosomeCluster | protein + energy | 蛋白+能量均衡 | +25% |
| **biofilmReactor** | **glucose + nitrogen + energy** | **三路均衡** | **+25%** |
| sporeSower | biomass + energy | 生物质+能量均衡 | +25% |
| nanoAssembler | qs + energy | QS+能量均衡 | +25% |
| pheromoneStation | protein + energy | 蛋白+能量均衡 | +25% |
| biomassConverter | protein + energy | 蛋白+能量均衡 | +25% |
| quantumExtractor | nitrogen + energy | 氮+能量均衡 | +25% |
| resonanceChamber | qs + energy | QS+能量均衡 | +25% |

**不受影响的建筑（单输入或无输入）**：
- glucoseCollector（无消耗）
- energyStation（只消耗 glucose）
- nitrogenFixer（只消耗 energy）
- energyBuffer（只消耗 glucose）
- transport / metabolicLoop（增益建筑）

→ **P1 阶段几乎不涉及**（只有 simpleExtractor 有双输入）
→ **P2 阶段开始有意义**（蛋白、DNA、氨基酸都是多输入）
→ **P3 生物膜反应器是同步系统的"明星建筑"**（三输入！）
→ **P4 全部建筑都受益**

**节奏上完美：渐进解锁同步玩法 = 渐进增加布局规划深度**

## 八、结论

### 一句话总结

> **直接移植 Factorio 的离散物品时序模型 = 重写半个游戏，不可行。
> 但用"供给同步加成"在速率模型上捕捉同样的设计精髓 = 90行代码，完美融合。**

### 三个系统的哲学

| 系统 | 奖励什么 | 乐趣来源 | 何时引入 |
|------|---------|---------|---------|
| **邻接** | 空间邻近 | "把对的东西放在旁边" | P1 起（被动、自然） |
| **传送带** | 网络连接 | "把远处的东西连起来" | P1 自动 → P2 手动 |
| **同步** | 供给均衡 | "让多路输入完美配合" | P2 起（主动、规划） |

三者叠加 = **一个完整的工程规划乐趣循环**：
1. 放对位置（邻接）
2. 拉好线路（传送带）
3. 调好比例（同步）

这正是你想要的"传送带连线的工程乐趣核心"。
