# 🎬 微生物帝国 UI/UX 视觉叙事诊断报告

## 一、逐问题诊断

### ❶ 建筑升级直观感不强（LV1-MAX 显示）

**现状分析：**
- 建筑升级后仅在左侧面板按钮右上角显示 `.act-tag`（小圆角标签 `Lv1`/`T1`/`MAX`），字号仅 `0.7em`
- 培养皿中的格子有 `.cell-tier` 角标（左上角），但 Lv1 时默认 **opacity:0 隐藏**，hover 才淡入
- 升级时虽有 `lvlUpFlash` 动画（0.6s金光），但**一闪即逝**，没有持久视觉差异
- **核心问题：升级前后的格子外观几乎没变化** — 同样的 emoji、同样的背景色、同样的边框

**严重程度：🔴 高**

> 打个比方：你花了一堆资源升级了建筑，结果它看起来跟没升之前一模一样，这感觉就像买了新衣服但穿的还是旧衣服。

---

### ❷ 文字信息过载

**现状分析：**
- 左侧面板 `380px` 宽，塞了 **资源 + 建造列表 + 科技树** 三大板块
- 每个建造按钮包含：图标 + 名称 + 描述 + 造价 + 等级标签 + 升级按钮 = **6个信息维度**
- 科技树节点包含：名称 + 描述 + 造价 + 前置条件 + 进度条 = **5个信息维度**
- 右侧面板 `280px` 宽，塞了 **目标 + 进化 + 变异 + 催化剂 + 流水线 + 统计 + 日志** = **7个板块**
- 所有文字使用等宽字体 `Share Tech Mono` 或 `Rajdhani`，行间距紧凑（0.7-0.88em）

**严重程度：🔴 高**

> 用户面对的不是"游戏"，而是一个**科研仪表盘**。信息量等同于一个复杂的运维监控面板。

---

### ❸ 参数变红/负载信号发现困难

**现状分析：**
- 资源产出变负时：`.res-rate.neg` 仅变 `--red` 颜色，**无动画、无图标变化、无声音提示**
- 建筑阻塞时：`.cell-blocked` 有红色脉冲边框 + 斜线条纹遮罩 + 顶部 "阻塞" 徽章，**这其实做得不错**
- 传送带阻塞：按钮变红 + 脉冲动画 `.has-blocked`，**有但不够醒目**
- **核心问题：** 
  - 关键警告信号分散在屏幕各处（左上资源区、中间格子、右侧状态）
  - 没有**统一的告警中心**或**视觉焦点引导**
  - 深色主题下，红色文字在深蓝背景上的对比度并不突出
  - 用户还没形成"扫哪里"的视觉习惯

**严重程度：🔴 高**

---

### ❹ 弹窗信息量过多

**现状分析：**
- 升级弹窗 `#upgradePopup`：名称 + 等级 + 效果说明 + 造价 = 4行文字 + 2个按钮
- 传送带升级弹窗 `#beltUpgradePopup`：同上 + 额外一行"传输效率影响产出" = 5行 + 2按钮
- 回收弹窗 `#recyclePopup`：名称 + 退还资源 + 2按钮
- 科技选择弹窗 `#choicePopup`：描述 + A/B两卡片各含标题+说明
- **快速升级面板** `#bldTypeSelector`：一次列出所有同类建筑，每个含 emoji+名称+等级+倍率+造价+升级按钮

**严重程度：🟡 中**

> 弹窗本身的信息量还算可控，但结合整个界面已经很密集的背景，每次弹窗都像在"信息轰炸"上再加一层。

---

### ❺ 上方/左右侧文字和参数过多

**现状分析：**
- **顶栏：** BIOSPHERE标题 + 版本署名 + 阶段徽章 + 进化提示 + 邻接规则提示 + 保存指示 + 不保存警告 + 加速 + 暂停 + 重置 + 音效 + 音乐 + 帮助 = **13个元素挤在一行**
- **帝国总览（培养皿上方）：** 核心展示 + 供能状态 + 升级条件 + 种群/食耗/功率/维护/供需/效率/物流/QS信号 + 菌群分工(3通道) + 成就/称号/昵称/总分/历史最高/排行榜 = **20+个数据点**
- **右侧面板：** 7个可折叠板块，每个都有数据
- **培养皿顶部操作提示：** "拖拽移动 · 双击升级 · 右键回收 · 框选批量 · 快捷键: 空格暂停 / 1-3加速 / ESC关闭"

**严重程度：🔴 高**

> 这就是你说的核心问题——**没有视觉层级（Visual Hierarchy）**。所有信息都在"喊"用户看它，但没有一个信息"更大声"。

---

### ❻ "菌主"角色代入感不足

**现状分析：**
- 除了 `🔮` 核心图标和"最初的生命痕迹...一团脆弱的原始液泡"描述文字外，**没有持续的角色提醒**
- 欢迎回来弹窗有"欢迎菌主回来！"（`🧫`）—— 仅此一处
- 游戏内没有角色头像、菌主形象、角色状态面板
- 建造行为描述是"建造碳源采集器"而非"指挥你的菌群去采集碳源"
- **Phase A 阶段色彩已实现**，但色彩变化比较微妙，用户未必能注意到"我的帝国在成长"

**严重程度：🟡 中**

> 微生物主题非常酷，但目前的感觉更像"操作工业模拟器"而非"统治一个微生物帝国"。

## 二、系统性方案设计

### 方案总纲："从仪表盘到生态缸"

当前UI的根本问题是**设计隐喻错了**——它在呈现一个"数据监控面板"，而不是"一个活生生的微生物世界"。

改造方向：**让数据退到背景，让生命感走到前台。**

---

### 🅰 方案 F：升级视觉分级系统（解决问题❶）

**目标：** 让建筑升级后**一眼就能看出区别**。

```
等级视觉映射：
  Lv1: 基础外观（当前样子）
  Lv2: 边框加粗1.5px + 微发光
  Lv3: 边框加粗2px + 呼吸光晕 + 角标常显
  Lv4: 渐变边框 + 持续微粒子 + 底纹变化
  MAX: 金色渐变边框 + 星星图标 + 持续光效 + 脉冲
```

**具体实现方向：**
1. **格子边框粗细递增**：`.cell.tier-2{border-width:1.5px}` → `.cell.tier-max{border-width:2.5px}`
2. **发光强度递增**：`box-shadow` 的 spread 和 blur 随等级增加
3. **emoji 尺寸微增**：高级建筑的 emoji 可以放大 5-10%
4. **背景纹理分级**：高级建筑增加微妙的渐变底纹
5. **升级瞬间**：增加一个 1.5s 的"进化波纹"动画（而非0.6s闪一下就没了）

### 🅱 方案 G：信息密度分层系统（解决问题❷❺）

**核心策略：渐进式信息披露（Progressive Disclosure）**

#### G1. 三层信息架构

```
第1层：核心摘要（始终可见，5秒内看完）

  - 当前阶段图标
  - 一句话当前目标

第2层：操作面板（需要时展开）
  - 建造列表（简化版：图标+名称+造价，描述hover展示）
  - 科技树（当前可用的才展开，未来的折叠）

第3层：详细数据（主动查看）
  - 完整统计
  - 日志
  - 成就殿堂
  - 详细资源收支
```

#### G2. 左侧面板简化方案

**建造按钮简化：**
```
当前: [图标] 碳源采集器 ×5          [T1] [⬆]
```

### 🅲 方案 H：智能告警系统（解决问题❸）

**核心思路：** 不要等用户发现问题，**主动把问题推到用户面前**。

#### H1. 统一告警条

在培养皿上方（帝国总览下方）增加一个**告警摘要条**：

```
┌─────────────────────────────────────┐
│ ⚠️ 2栋建筑阻塞  📉 葡萄糖-1.00/s  │
└─────────────────────────────────────┘
```

- 没有问题时不显示（零噪音）
- 有问题时自动滑入，红色/橙色醒目显示
- 点击可快速定位到问题建筑

#### H2. 资源负值强化

```css
/* 当前：仅红色文字 */
.res-rate.neg { color: var(--red); }

/* 增强：红色 + 脉冲 + 向下箭头 */
.res-rate.neg {
  color: var(--red);
  animation: resNegPulse 1.5s ease-in-out infinite;
}
.res-rate.neg::before { content: '▼ '; }
```

#### H3. 格子健康指示器

在格子底部增加一个 **3px 高的健康条**：
- 🟢 绿色 = 正常工作
- 🟡 黄色 = 输出被限制
- 🔴 红色 = 阻塞/无资源

## 四、总结

> **一句话诊断：** 你的游戏拥有精心设计的深度系统和出色的视觉叙事增强（Phase A-E），但UI正在试图"一次展示所有东西"，导致**信息密度超过了人类注意力的极限**。

> **一句话方案：** 把"所有数据都展示给用户看"改为"只在用户需要时才展示"，同时用视觉手段（颜色、动画、大小）替代文字手段来传递状态信息。

> **核心公式：** 少文字 + 多视觉 + 渐进披露 + 角色代入 = 从"监控面板"变成"生态缸"
