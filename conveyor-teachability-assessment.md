# 🎓 传送带同步系统 — 玩家可理解性评估

## 核心问题

> "如果加了这种改动，玩家能理解吗？怎么知道这样操作才更好？"

这是关卡设计里最关键的问题之一。**你不用教什么，你需要教什么，什么时候教**——这三层决定了新系统是"深度乐趣"还是"复杂负担"。

---

## 1. 当前游戏已有的教学体系分析

### ✅ 已经做得很好的

| 教学手段 | 位置 | 效果评级 |
|---------|------|---------|
| **GUIDE 引导条** | 顶部常驻 | ⭐⭐⭐⭐ 条件化逐步引导，非常清晰 |
| **阶段性解锁** | P1→P5渐进 | ⭐⭐⭐⭐ 不一次性暴露所有复杂度 |
| **P1自动连线** | P1阶段 | ⭐⭐⭐⭐⭐ 新手零负担，先看到再操作 |
| **P2手动连线提示** | `_showBeltGuide()` | ⭐⭐⭐ 放置建筑后弹框引导连接 |
| **传送带tooltip** | 悬停显示 | ⭐⭐⭐ 效率%、容量、距离 |
| **无效连接警告** | 橙色弹窗 | ⭐⭐⭐⭐ 连错立刻有反馈 |
| **邻接脉冲光波** | Canvas实时 | ⭐⭐⭐⭐⭐ 视觉上"看得到加成" |

### ❌ 当前缺失的（如果加同步系统会暴露的问题）

| 缺失点 | 为什么是问题 |
|--------|------------|
| **没有"为什么这样更好"的实时反馈** | tooltip只显示当前状态，不显示"如果你这样做会更好" |
| **缺少数值变化的因果归因** | 产出数字变了，但玩家不知道是邻接、传送带还是科技导致的 |
| **没有视觉上的"满意度/健康度"指标** | 邻接有光波，但传送带网络没有整体状态指示 |
| **GUIDE引导没有覆盖"优化"层** | 引导条教"建什么"和"连什么"，但不教"怎么连更好" |

## 2. 同步加成的教学设计方案

### 核心原则：**先感受，再数字，最后理论**

这是关卡设计的黄金法则——
- **第一层：感受** → 玩家看到视觉变化（"这条线变亮了！"）
- **第二层：数字** → 玩家看到数据变化（"+15% 同步加成！"）
- **第三层：理论** → 玩家理解原理（"原来多路均衡供给才能最优"）

很多游戏犯的错是直接从第三层开始——弹一面墙的文字教程。**永远不要这样做。**

---

### 2.1 第一层教学：视觉直觉（零认知成本）

#### 方案：传送带颜色温度变化

**现状**：所有正常传送带都是相同的工业风视觉，只有无效的才变灰色。

**改进**：根据多输入建筑的"供给均衡度"，传送带和建筑自动变色——

```
供给均衡度     传送带视觉                    建筑视觉
─────────    ──────────────────         ──────────────────
0% 严重失衡   暗红微光 + 货物粒子变慢        格子边框微红闪烁
30% 部分失衡  暖黄色调 + 正常速度            格子正常
70% 接近均衡  青绿色调 + 货物加速             格子微微发绿
100% 完美同步 金色流光脉冲 ✨ + 最快速度      格子金色边框 + 粒子爆发
```

**为什么有效**：
- 玩家不需要看任何数字就能直觉感知："这边红了，那边绿了"
- 类似交通信号灯——红黄绿是人类本能理解的色彩语义
- **邻接系统已经用了这种方法**（脉冲光波的亮度=加成强度），所以视觉语言一致

**代码改动量**：~25行（在 `drawConveyorBelts` 中根据同步度修改 `mainColor` 的混合比例）

---

### 2.2 第二层教学：数字浮标（低认知成本）

#### 方案：同步达成瞬间弹出浮动文字

**触发时机**：当多输入建筑的同步度**首次**从低于70%变为高于70%时——

```
          ╭─────────────────────╮
          │  ⚡ +15% 供给同步！  │
          │  (三路原料均衡供给)   │
          ╰─────────────────────╯
                    ↑
            [生物膜反应器格子]
```

**关键细节**：
- **只在阈值跨越时弹出**，不是每帧刷新——避免信息疲劳
- **浮动文字2秒后淡出**，不干扰正常游玩
- **显示加成来源**："供给同步"四个字就够了
- 如果同步度又掉下去了，用不同颜色提示："⚠ 供给失衡 -10%"

**为什么有效**：
- 建筑变化 → 数字跳出 → 因果关系一目了然
- "我刚连了一条带，数字就跳了" → 玩家建立操作-反馈链路
- 符合你游戏已有的 `showCursorTooltip()` 和 `buildBurst()` 反馈模式

**代码改动量**：~20行（在 `updateRates()` 中检测同步度变化，触发浮动文字）

---

### 2.3 第三层教学：引导系统集成（主动教学）

#### 方案A：GUIDE引导条的精准时机触发

在 P3 阶段的 GUIDE 里加一条——恰好在**生物膜反应器**这个三输入建筑出现的时候：

```javascript
// GUIDE[3] 新增引导步骤
{
  check: s => s.bldCount('biofilmReactor') > 0 && s.getSyncBonus('biofilmReactor') < 0.5,
  text: '⚡ 生物膜需要三路原料！让葡萄糖、氮源、能量的传送带都充裕且均衡 → 触发供给同步加成！',
  icon: '🔄'
}
```

**为什么选 P3 生物膜反应器**：
- 它是游戏中第一个**三输入建筑**（葡萄糖+氮源+能量）
- P1/P2的建筑最多两个输入，同步效果不明显
- 这是一个自然的"教学节拍"——玩家刚学会手动连线（P2），现在该学"连得更好"

#### 方案B：传送带tooltip增加同步信息

**现有tooltip**：
```
⛓ 传送带 Lv.2
🧱 生物膜反应器 ← ⚡ ATP合成酶
⚡ 传输效率: 90%  · 📦 容量: 3/s
📍 紧邻（最佳）
```

**改进后tooltip**（仅对多输入建筑的传送带显示额外一行）：
```
⛓ 传送带 Lv.2
🧱 生物膜反应器 ← ⚡ ATP合成酶
⚡ 传输效率: 90%  · 📦 容量: 3/s
📍 紧邻（最佳）
🔄 供给同步: 72% (+11%产出)  ← 新增！
💡 三路原料越均衡，加成越高！  ← 新增！
```

**关键**：这一行**只在多输入建筑的传送带**上才出现。单输入建筑不显示，避免信息过载。

**代码改动量**：~15行（在悬停tooltip渲染中追加判断）

### 2.4 教学时间线总览

```
P1（新手期）          P2（进阶期）         P3（核心教学点）        P4-P5（精通期）
─────────────       ──────────────     ─────────────────     ──────────────
自动连线              手动连线             同步系统首次出现         深度优化
玩家只看到带在跑      玩家学会连线操作      生物膜 = 三输入          追求满同步
                                       ↓                      ↓
不需要教什么           引导条教连线操作     ① 视觉变色（感受）      成就驱动
                    无效连接有警告        ② 浮标数字（反馈）      完美同步追求
                                       ③ 引导条提示（理论）
                                       ④ tooltip确认（验证）
```

**这就是关卡设计的核心：在玩家需要的时候，用最小信息量教授。**

## 3. 与上次方案 A/B/C/F 的教学整合

### 统一的视觉语言系统

| 系统 | 颜色语义 | 视觉手段 |
|------|---------|---------|
| 邻接 | 规则类别色（蓝/绿/紫/金/红） | 边缘脉冲光波 |
| 传送带连接 | 资源色（绿/蓝/黄） | 工业轨道 + 货物粒子 |
| 供给同步 | 红→黄→绿→金（温度渐变） | 传送带色调变化 + 格子边框 |
| 连击反馈 | 青→金（连击数越高越金） | 爆发粒子 + ×N浮标 |

**所有视觉反馈都是颜色驱动的**——玩家只需要一个直觉："越金越好"。

### 信息分层原则

```
第0层（背景）  → 传送带在跑（有东西在流动）      ← 已有
第1层（直觉）  → 颜色在变（红的不好，绿的好）    ← 方案F视觉层
第2层（数字）  → 浮标弹出（+15%同步！）          ← 方案A+F结合
第3层（详情）  → tooltip完整数据                  ← 方案B扩展
第4层（成就）  → 🏆首次同步！🏆完美同步！         ← 方案C扩展
第5层（引导）  → GUIDE引导条提示                  ← 方案F教学层
```

**每层都是可选的**。玩家只看第0-1层就能玩好，想深入的才去看第2-5层。

## 4. 潜在的理解障碍和应对

### 障碍1："同步"这个概念太抽象

**应对**：不用"同步"这个词。用**"供给均衡"**或直接用颜色——
- ✅ "三路原料均衡供给！+15%"
- ❌ "传送带时序同步达成，多输入建筑供给链充裕度加权均值超过阈值"

### 障碍2：玩家不知道哪条输入不足

**应对**：在多输入建筑的格子上，hover时显示每条输入的"充裕度柱状图"——

```
┌─ 生物膜反应器 ─────────┐
│ 🟢 葡萄糖  ████████░░  80%  │
│ 🔵 氮源    ██████░░░░  60%  ← 瓶颈！│
│ ⚡ 能量    █████████░  90%  │
│                              │
│ 🔄 供给均衡: 60%  (+8%)      │
│ 💡 氮源不足！多造固氮装置      │
└─────────────────────────────┘
```

**这就是"空间中的故事"**——玩家一看就知道问题在哪，解决方案是什么。

### 障碍3：和邻接系统混淆

**应对**：
- 邻接 = **边缘**光波（在两个格子之间）
- 同步 = **格子内部**颜色变化 + 传送带色调
- 视觉位置完全不同，不会混淆
- tooltip里分开两行显示："邻接: +20%  |  供给同步: +15%"

### 障碍4：信息过载（最大风险）

**应对**：**渐进式信息密度**——
- P1-P2：同步系统**完全隐藏**（代码里直接跳过计算）
- P3首次出现：只有视觉（颜色）+ 一次引导条提示
- P4+：tooltip、成就、浮标全部激活
- 转生后：玩家已理解，全部系统从P2就开启

## 5. 最终结论

### 可理解性评级：⭐⭐⭐⭐☆（4/5，高可行性）

**前提条件**（缺一不可）：
1. ✅ 视觉先行——颜色变化必须在数字之前让玩家感受到
2. ✅ P3生物膜反应器作为教学锚点——不提前暴露
3. ✅ 不用"同步"这个术语——用"供给均衡"
4. ✅ 多输入建筑hover显示每条输入的充裕度——让瓶颈可见
5. ✅ 与邻接系统的视觉位置分离——避免混淆

**扣的1分**：
- 对极度休闲的玩家来说，多了一个需要关注的指标。但因为P1-P2完全隐藏，影响可控。

### 推荐的实施优先级

```
必做（Day 1）
  ① 传送带颜色温度变化（~25行）— 零认知成本的视觉直觉
  ② P3引导条新增一条提示（~5行）— 精准时机教学

应做（Day 2）
  ③ 同步达成浮动文字（~20行）— 操作-反馈链路
  ④ tooltip增加供给同步行（~15行）— 验证和确认

可选（Day 3）
  ⑤ 多输入建筑hover充裕度柱状图（~40行）— 深度诊断
  ⑥ 成就扩展"首次同步""完美同步"（~15行）— 长期驱动
```

**总代码量：~120行**（Day 1只需~30行）

## 6. 一句话总结

> **不是教玩家"同步系统是什么"，而是让玩家自己发现"这样连更好"。红变绿，数字跳，成就弹——三层反馈，零面墙文字。**
