# 🧬 微生物帝国 — 动机引擎设计方案

> **核心问题**：玩家完成初始引导后，缺少持续游玩动机。"我为什么要玩这个？" "我为什么要一直放建筑？"
>
> **版本**：v2.1.1 | **日期**：2026-03-22 | **状态**：方案阶段
>
> **v2.0 更新**：新增改动 G/H/I — 格子渐进解锁 + 生物膜边界 + 引导就近指引
> **v2.1 更新**：新增改动 J — 帝国核心入格 + 供能链接可视化 + 核心视觉渐进生长
> **v2.1.1 补丁**：格子居中定位 + 帝国总览精简 + 引导偏移量定位 + 视觉锚点稳定性

---

## 一、问题诊断

### 1.1 症状

| 体验节点 | 当前状态 | 问题 |
|----------|---------|------|
| 开场（0-10s） | tab弹窗介绍游戏目标和阶段路线 | **信息过载** — 玩家还没有任何情感投入就被灌输5个阶段的规划 |
| 初见网格 | 20×20=400格全部展开 | **空旷压迫** — 满屏空格子让玩家觉得"我得填满这些"，无从下手 |
| 帝国核心 | 独立HUD状态条在网格上方 | **认知断裂** — 核心与格子在视觉上完全分离，"核心供能"只是一行文字而非可感知的关系 |
| 引导（10-60s） | "建造碳源采集器""建造ATP合成酶"… | **操作清单** — 只告诉"做什么"，没回答"为什么" |
| 引导位置 | 左上角dish-guide-bar | **远离战场** — 玩家注意力在网格中心，引导在视野边缘 |
| 早期循环（1-5min） | 放建筑→看数字涨→攒资源→放更多建筑 | **行为无意义感** — 放第1个和放第8个体验几乎一样 |
| 中期过渡（P1→P2） | 探索度任务清单 + 进化按钮 | **目标太功能化** — "建3条传送带"没有叙事包装 |
| 空间感知 | 20×20网格始终不变 | **缺少成长感** — 帝国在"壮大"但视觉空间没有变化 |
| 格子定位 | 格子顶部对齐(align-content:start)，帝国总览在格子上方 | **视觉锚点不稳** — 帝国总览展开/折叠时格子盘上下跳动，核心入格后焦点漂移更明显 *(v2.1.1)* |

### 1.2 根因分析（v2.1 更新）

```
玩家动机 = 情感锚点 × 行为意义 × 好奇心引擎 × 空间成长感
         = 0.2      × 0.4       × 0.5         × 0.1
         = 远远不够

空间成长感（0.1/1.0）：
  - 20×20一次性给满，空间没有"解锁"的概念
  - 放5个建筑 vs 放50个建筑，网格看起来一样大
  - 缺少"我的领地在扩张"这个最直观的成长隐喻
  
核心认知断裂（v2.1新增）：
  - 帝国核心在UI顶部状态条里，不在游戏世界中
  - "核心供能2台"是阅读理解题，不是直觉体验
  - 核心与建筑之间缺少"物理连接"的视觉表达
```

### 1.3 核心结论

> **不是游戏不好玩，是游戏没有在前30秒告诉玩家它的灵魂是什么。**
> **而且400格空地上来全给，就像把整张世界地图一次性摊开——没有探索的冲动了。**
> **帝国核心——这个帝国的"心脏"——应该活在格子世界里，而不是藏在HUD状态条里。** *(v2.1新增)*

## 二、设计理念

### 2.1 四层动机模型（v2.1 继续使用四层）

```
┌─────────────────────────────────────────┐
│  第四层：空间成长感                       │  "我的帝国在扩张"
│  格子解锁 · 领地扩张 · 膜的生长           │
│  核心生长 · 供能网络扩展 (v2.1)            │
├─────────────────────────────────────────┤
│  第三层：好奇心引擎                       │  "下一个是什么？"
│  渐进揭示 · 阶段预告 · 悬念设计            │
├─────────────────────────────────────────┤
│  第二层：行为意义                         │  "我做的事情有意义"
│  空间叙事 · 啊哈时刻 · 里程碑可视化         │
│  供能链接 = 行为的即时反馈 (v2.1)           │
├─────────────────────────────────────────┤
│  第一层：情感锚点                         │  "我在乎这个世界"
│  叙事开场 · 角色共情 · 世界观浸润            │
│  核心=心脏=生命之源 (v2.1)                 │
└─────────────────────────────────────────┘
```

### 2.2 设计原则

1. **先打动，再教学** — 开场先建立情感，再教操作
2. **行动即叙事** — 每次放建筑都是在讲述一个微生物进化的故事
3. **30秒法则** — 第一个"啊哈时刻"必须在30秒内出现
4. **始终有迷雾** — 永远让玩家看到一点点他还不能触及的东西
5. **空间即奖励** — 格子是有限资源，扩张是最直观的成长 *(v2.0)*
6. **看见即理解** — 核心供能、邻接加成等规则应该用视觉而非文字传达 *(v2.1新增)*

## 三、方案详情

---

### ★ 改动 J：帝国核心入格 + 供能链接可视化（v2.1 核心新增）

> **帝国核心从HUD状态条迁移到网格中心，成为游戏世界中有物理存在的"心脏"。放下碳源采集器时，一条能量链接从核心伸出——你能看到能量在流动。**

#### J1. 核心入格 — 占据中心1格

```
核心永远占据网格正中心的1个格子：

  3×3 (P1):     5×5 (P2):          7×7 (P3):
   □ □ □        □ □ □ □ □        □ □ □ □ □ □ □
   □ 🔮 □       □ □ □ □ □        □ □ □ □ □ □ □
   □ □ □        □ □ 🔮 □ □       □ □ □ □ □ □ □
                □ □ □ □ □        □ □ □ 🔮 □ □ □
                □ □ □ □ □        □ □ □ □ □ □ □
                                 □ □ □ □ □ □ □
                                 □ □ □ □ □ □ □

核心格特性：
  - 不可拆除、不可移动、不可放置其他建筑
  - 点击核心格 → 弹出核心升级面板（复用现有功能）
  - 视觉上比普通格子略大（transform: scale(1.05)）或有独特边框
  - 核心格始终显示当前阶段的核心emoji和名称

格子数影响：
  P1: 3×3=9格, 核心占1格 → 实际可用8格（足够P1的4种建筑）
  P2: 5×5=25格, 核心占1格 → 实际可用24格
  后续同理，核心永远只占1格
```

#### J2. 供能链接 — 核心到碳源采集器的能量丝

```
当碳源采集器放置时，能量链接动画序列：

0.0s — 建筑放置确认（现有cellDivision动画）
0.3s — 核心闪烁一下（感应到新的"子体"）
0.5s — 一条半透明能量丝从核心格"射出"
       丝不是硬直线，是有机弯曲的曲线（类似微管/菌丝）
       丝的颜色 = 当前阶段色（Phase Color），50%透明度
0.8s — 能量丝连接到碳源采集器格子
       连接瞬间，采集器格子闪亮一下（"供能接通"）
1.0s — 进入持续供能状态
       微小的光点沿能量丝从核心流向采集器
       流速与采集器产出频率同步（约每0.5s一个光点）

能量丝的视觉细节：
  - 宽度：2-3px
  - 曲线：贝塞尔曲线，有轻微的自然摆动（0.5s周期微动画）
  - 路径：不是直连，而是沿格子边缘走的"L形"或"弯曲路径"
  - 多条丝互不重叠（自动选择不同路径）
```

#### J3. 供能状态的视觉反馈

```
空闲状态（0/2台已连接）：
  - 核心缓慢搏动，光芒较弱
  - 没有能量丝
  - 核心似乎在"等待"

部分供能（1/2台已连接）：
  - 1条能量丝活跃
  - 核心搏动稍快
  - 核心光芒增强

满载供能（2/2台已连接）：
  - 2条能量丝同时活跃
  - 核心搏动最快，脉冲光
  - 核心格子边缘有持续光晕

超载尝试（试图放第3台碳源采集器）：
  - 核心闪烁红光 0.3s
  - 能量丝抖动一下
  - 提示气泡："🔮 核心供能已满(2/2) — 进化后可供能更多"
  - 建筑仍然可以放置，但不享受免费供能（需自行消耗ATP）
  - 无核心链接的碳源采集器有一个"断连"视觉标记（暗色/虚线框）
```

#### J4. 核心视觉渐进生长 — 1格不变，"气场"递进

```
核心永远占1格，但视觉表现随阶段升级"生长"：

P1 — 原始液泡 🔮
  光效：微弱的绿色脉动，几乎不溢出格子边界
  搏动：4s周期，幅度小
  能量丝：细（1-2px），颤抖感，像初生的触手
  整体印象："一团脆弱的、刚刚苏醒的东西"

P2 — 原核聚落 🦠
  光效：明亮的绿色，光芒开始溢出到邻近格子的边缘
  搏动：3s周期，幅度增大
  能量丝：变粗（2-3px），流速更快，更稳定
  新增：核心格周围8格的边缘有极微弱的绿色光泽（辐射感）
  整体印象："它站稳了脚跟，开始有力量了"

P3 — 生物膜枢纽 🔬
  光效：蓝色光晕，扩散覆盖周围一圈格子
  搏动：2.5s周期，有"呼吸"节奏感
  能量丝：蓝色，带分叉，像微管网络
  新增：核心格有持续的微粒子效果（2-3个蓝色光点绕核心漂浮）
  整体印象："一个复杂的枢纽，信息在流转"

P4 — 菌落中枢 🏗️
  光效：金色辐射，覆盖周围两圈格子
  搏动：2s周期，脉冲式（"心跳"节奏）
  能量丝：金色蛛网状，所有连接的建筑都有微弱金光
  新增：周围被供能的建筑有"受供"光泽（淡金色overlay）
  整体印象："帝国的心脏，强有力地搏动"

P5 — 生命圣殿 ✨
  光效：紫色耀光脉冲，整个网格被微弱光辉笼罩
  搏动：1.5s周期，配合粒子喷发
  能量丝：紫色光束（不是丝了，是"束"），稳定耀眼
  新增：核心有持续的粒子喷发效果（星尘感）
        整个培养皿的培养基纹理在核心光照下微微发亮
  整体印象："不是一个建筑，是一个微型太阳"
```

#### J6. 技术实现要点

```
涉及文件：game.js, index.html

核心入格改动：
  1. 初始化时在网格中心放置核心（grid[centerIdx] = { type: '_coreColony' }）
  2. renderGrid() 中核心格特殊渲染（大emoji + 阶段色背景 + 光效CSS类）
  3. cellClick() 中核心格拦截 → 弹出核心面板（复用现有逻辑）
  4. 建筑放置/移动/框选等操作跳过核心格

供能链接绘制：
  方案A（推荐）— SVG叠加层：
    在网格容器上叠加一个等大的 <svg> 元素
    每条供能链接 = 一个 <path> + 沿路径移动的 <circle> 粒子
    贝塞尔曲线生成算法：根据核心和目标的相对位置计算control points
    光点动画：SVG <animateMotion> 或 SMIL / CSS offset-path
    
  方案B — Canvas叠加层：
    requestAnimationFrame 绘制曲线和粒子
    更灵活但性能开销稍大

核心光效：
  CSS实现（推荐）：
    .core-cell { --pulse-speed: 4s; --glow-radius: 0px; }
    .core-cell::after {
      content: '';
      position: absolute;
      inset: calc(-1 * var(--glow-radius));
      border-radius: 50%;
      background: radial-gradient(circle, var(--phase-color-20) 0%, transparent 70%);
      animation: corePulse var(--pulse-speed) ease-in-out infinite;
      pointer-events: none;
    }
  JS负责更新CSS变量：随阶段变化更新 --pulse-speed 和 --glow-radius

原有HUD处理：
  - .core-colony-bar 不直接删除，改为简化版"核心状态指示器"
    （小型，显示供能 X/Y，点击跳转到核心格）
  - 或完全移除，所有信息通过核心格交互获取

存档兼容：
  - 旧存档没有核心格 → load() 时自动在中心放置
  - 如果中心格已有建筑 → 将该建筑移到最近的空格子

CORE_COLONY 数据扩展：
  const CORE_COLONY = {
    1: { ...现有字段,
         pulseSpeed: 4, glowRadius: 0, linkStyle: 'tendril', particleCount: 0 },
    2: { ...现有字段,
         pulseSpeed: 3, glowRadius: 1, linkStyle: 'tendril', particleCount: 0 },
    3: { ...现有字段,
         pulseSpeed: 2.5, glowRadius: 1, linkStyle: 'tube', particleCount: 3 },
    4: { ...现有字段,
         pulseSpeed: 2, glowRadius: 2, linkStyle: 'web', particleCount: 5 },
    5: { ...现有字段,
         pulseSpeed: 1.5, glowRadius: 3, linkStyle: 'beam', particleCount: 8 },
  };

工时估：
  J1 核心入格基础：1.5h（格子渲染+点击拦截+面板迁移）
  J2 供能链接SVG：2-3h（曲线计算+粒子动画+多链接管理）
  J3 状态反馈：0.5h（空闲/部分/满载/超载 四种状态切换）
  J4 视觉渐进：1-1.5h（5阶段CSS变量+光效+粒子）
  J5 原有HUD迁移：0.5h
  总计：6-8h
```

### ★ 改动 G：格子渐进解锁（v2.0 核心新增）

> **这是整个方案中改变"骨架"的改动——从"填满400格"变成"培育一个不断生长的有机体"。**

#### G1. 阶段-格子映射（v2.1 更新：核心占1格）

| 阶段 | 网格尺寸 | 格子数 | 核心占用 | 实际可用 | 设计理由 |
|------|---------|--------|---------|---------|---------|
| P1 原始液泡 | 3×3 | 9 | 1 | **8** | 8格+核心，前4种建筑刚好够体验。邻接从第一秒就重要 |
| P1→P2 过渡 | 5×5 | 25 | 1 | **24** | 第一次扩张，膜破裂→重组动画，叙事："你的菌落突破了原始边界" |
| P2 代谢觉醒 | 5×5 → 7×7 | 25→49 | 1 | **48** | P2中期达到探索度阈值时再次扩张，奖励感 |
| P2→P3 过渡 | 9×9 | 81 | 1 | **80** | 进入物流时代，需要更大空间布局传送带网络 |
| P3 物流帝国 | 9×9 → 11×11 | 81→121 | 1 | **120** | P3中期扩张，菌丝网络覆盖更广 |
| P3→P4 过渡 | 13×13 | 169 | 1 | **168** | 群体意识需要更密集的信号网络 |
| P4→P5 过渡 | 15×15 | 225 | 1 | **224** | 终局大扩张 |
| P5+ / 转生后 | 最大 20×20 | 400 | 1 | **399** | 转生后可选择更大网格作为转生奖励 |

> **注意**：以上数字是初始设计值，需要根据各阶段建筑数量和玩家实际填充率调试。核心原则是**每次扩张时玩家已经基本填满当前格子（70-80%占用率）**。核心占1格不影响填充率计算（核心格不计入可用格子）。

#### G2. 扩张触发机制

```
扩张不是自动的，需要玩家"感受到"：

方案A（推荐）—— 绑定阶段进化：
  触发：_doPhaseUpgrade() 中，在仪式序列完成后执行 _resizeGrid()
```

## 六、不做什么

明确排除以下方向（避免范围蔓延）：

- ❌ **不加长篇剧情/对话系统** — 放置游戏的叙事应该是环境叙事，不是RPG对话
- ❌ **不重构核心玩法循环** — 现有机制（邻接/传送带/同步）是好的，问题在包装不在内核
- ❌ **不增加新的游戏系统** — 不加任务系统/日常系统/成就重做，这些都是后期优化
- ❌ **不做不规则扩张形状** — 先用方形，虽然有机体形状更酷，但实现复杂度不值得现阶段投入
- ❌ **不做格子购买/付费解锁** — 扩张是进化的自然结果，不是交易
- ❌ **核心不占多格** — 核心永远1格，通过光效"气场"表达成长，不吃玩家空间 *(v2.1新增)*
